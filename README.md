# EJ-RG-2
En este proyecto se realizo con el fin de realizar practica del uso de markdown. con el fin de hacer un readme que sirva como guia para que realizaen proyectos de software.
## Instalación y ejecución
1. Crearemos un nuevo repositorio en GitHub desde la interfaz web.
2. Verificamos dentro de GitHub su creacion y los permisos correrspondientes.
3. Abriremos el archivo README.md para su edición.
4. Editamos el archivo README de acuerdo a la sintaxis Markdown.

## Características
- Ejemplo de estructura básica de README
- Explicaciones claras y organizadas
- Incluye comandos y fragmento de código
- Fácil de personalizar para otros proyectos
